# 🗓️ meetfit — 복잡한 맥락을 계산해 정답을 알려주는 회의 예약 솔루션

> **2026 Toss Product Design Challenge** > "6명의 동료가 1시간 회의 일정을 잡는 경험을 설계해주세요."

많은 사람이 일정을 잡을 때 기존 캘린더 앱은 단순히 '비어 있는 시간'만 보여줍니다. 하지만 실제 협업 환경에서는 필수/선택 참석자 구분, 점심 직후 피로도, 외근 이동 시간 등 **눈에 보이지 않는 개인의 맥락을 사람이 수동으로 계산**해야 합니다.

`meetfit`은 무겁고 번거로운 수동 스케줄러 페이지를 과감히 제거하고, 연동된 데이터와 주관적 성향 프리셋을 계산하여 **모두에게 가장 쾌적한 최적의 의사결정 시간대를 점수로 제안**하는 툴입니다.

---

## ✨ 핵심 기능 및 화면 설계 (Core UX)

### 01. 회의 프로필 프리셋 
* 사용자가 매번 번거롭게 캘린더에 반복 일정을 그릴 필요가 없습니다. 기본 사내 캘린더(Google, Outlook) 데이터가 실시간 연동됩니다.
* 사용자는 캘린더가 알 수 없는 **인간의 주관적 선호 맥락(집중 업무 시간, 기피 시간, 선호 시간)**만 탭 몇 번으로 가볍게 등록해 둡니다.
* 이름·소속팀·프로필 사진도 이 화면에서 직접 확인하고 수정할 수 있어, 회의 추천뿐 아니라 '나'라는 사람을 나타내는 정보까지 한곳에서 관리합니다.

### 02. 적합도 스코어링 시스템
* 기존 서비스의 흑백논리(가능/불가능)에서 벗어나, 현실의 '회색 지대(되긴 하는데 피곤한 시간 등)'를 **정량화된 점수**로 치환하여 인지 부하를 줄였습니다.
* **회의 전후 여유 시간 (피로도 제어):** 추상적인 피로도 개념 대신, '회의 앞뒤로 최소 1시간의 빈 블록이 확보되었는가'를 정량 규칙으로 삼아 점수에 반영하고, 직관적인 카피로 근거를 노출합니다.
* **외근 전후 이동 버퍼:** 캘린더 연동으로 들어온 외근 일정은 시작 전·종료 후 각 최소 1시간을 회의 불가능한 시간으로 자동 제외해, "외근 가기 직전/막 돌아온 직후에 회의가 잡히는" 비현실적인 추천을 원천적으로 막습니다.
* **동점이면 더 빠른 시간 우선:** 여러 후보의 적합도 점수가 동일하다면, 굳이 미룰 이유가 없다는 원칙에 따라 더 이른 시간을 우선 추천합니다. '가장 잘 맞는 시간'이 여러 개일 때 그중 가장 빠른 하나로 자동 수렴시켜, 사용자가 추가로 고민할 필요가 없게 했습니다.
* **참석자별 회의 적합도 히트맵:** 실제로 이 회의에 추가된 참석자만 보여주고, 셀 하나하나의 색(좋음/가능/어려움)과 클릭했을 때 나오는 점수·사유가 항상 일치하도록 만들어 '근거를 눌러봤는데 앞뒤가 안 맞는' 신뢰도 저하 요소를 없앴습니다. 추천(★) 시간은 항상 참석자 전원에게 가장 좋은 조건으로 표시됩니다.

### 03. 주간 기간 우선 추천 디폴트
* 실무에서 6명의 일정을 잡을 때 특정 일을 고집하기보다 "다음 주 중 편한 시간"으로 접근하는 맥락이 더 큽니다.
* 일자별 탐색 대신 **[기간으로 찾기]**를 디폴트로 하여, 일주일 치 데이터 중 모두가 가장 쾌적한 '최적의 하루와 1시간 슬롯 TOP 3'를 먼저 스포트라이트 형태로 보여줍니다.

### 04. 필참/선택 참석자의 역할 분리
* 필참자는 후보 시간 중 **가능한 시간을 골라 합의에 참여**하지만, 선택 참석자는 시간 결정에 관여하지 않는다는 원칙을 그대로 인터랙션에 반영했습니다.
* 그래서 선택 참석자에게는 "이 후보들 중 골라주세요"가 아니라, **이미 확정된 시간에 참석 여부만** 묻습니다. 홈 화면의 "나에게 온 제안"에서 필참 초대·선택 참석 초대·(확정 후) 취소 알림까지 세 시나리오를 각각 시뮬레이션해볼 수 있습니다.

### 05. 제안 수정/취소 정책
* 실무에서 회의 제안은 한번 응답이 들어오기 시작하면 조건을 바꾸는 순간 그 응답이 무효가 되는 문제가 있습니다.
* 그래서 **아직 아무도 응답하지 않았을 때만 "제안 수정하기"**를 허용하고, 응답이 하나라도 들어온 뒤에는 수정 버튼을 비활성화한 채 "제안 취소하기"만 남겨 둡니다 — 비즈니스 로직을 화면 상태로 직접 드러낸 예시입니다.

### 06. 참석 현황 진행률 + 리마인드
* 필참자 의견 수집 화면과 회의 상세의 선택 참석자 응답 현황 모두 **"몇 명이 답했는지"를 진행률 바 하나로 즉시 파악**할 수 있게 했습니다.
* 미응답자가 남아 있으면 리마인드를 보낼 수 있어, 조율이 멈추지 않고 계속 앞으로 나아가도록 설계했습니다.

### 07. 회의 장소(선택) 기록
* 온라인/오프라인 여부처럼 사소하지만 매번 놓치기 쉬운 정보를 회의 생성 단계에서 선택 입력으로 가볍게 남기고, 상세 화면과 초대 카드에 그대로 노출합니다.

## 🧭 실제 사용자 플로우

### 회의를 새로 잡을 때
1. **회의 정보 입력** — 제목·장소(선택)를 적고 참석자를 추가한 뒤, 참석자마다 **필참/선택**을 지정합니다.
2. **회의 목적 선택** — 정보 공유·의견 수렴·의사결정·문제 해결·긴급 대응 중 하나를 고르면 점수 가중치가 그에 맞게 바뀝니다. 똑같은 참석자·똑같은 후보 시간이라도 "왜 모이는지"에 따라 다른 시간이 추천될 수 있습니다.
3. **탐색 범위 지정 → 추천 실행** — **기간으로 찾기**(디폴트) 또는 특정 날짜 지정 중 골라 "MeetFit 추천받기"를 누르면, 최대 3개의 후보가 TOP 3 스포트라이트로 뜹니다.
4. **추천 근거 확인** — 카드를 펼치면 **적합도 계산 기준**(막대 그래프)과 **참석자별 회의 적합도 히트맵**을 볼 수 있어요. 히트맵은 이 회의에 실제로 추가된 사람만 보여주고, 셀 하나를 누르면 그 시간·그 사람만의 점수와 사유가 뜹니다 — 색과 설명이 항상 일치합니다.

### 참석자 의견을 모을 때
5. **제안 발송** — **필참자에게 제안하기**를 누르면 홈 화면의 "제안 중인 회의"에 뜨고, 각 필참자 입장에서 응답을 시뮬레이션해볼 수 있습니다.
6. **수정/취소 정책** — 응답이 하나도 안 들어온 상태라면 **제안 수정하기**(제목·참석자·목적을 그대로 이어받아 다시 조율)가 가능하고, 응답이 하나라도 들어오면 그때부터는 **제안 취소하기**만 남습니다.
7. **합의 확정** — 필참자 응답이 모이면 **합의 화면**에서 최종 후보를 확정합니다.

### 확정된 회의
8. **참석 여부 확인** — 확정 즉시 선택 참석자에게 참석 여부를 묻고, **회의 상세** 화면에서 응답 현황을 진행률 바로 계속 확인하고 리마인드를 보낼 수 있습니다.
9. **일정 변경/취소** — **일정 변경하기**는 제목·장소·참석자·목적을 그대로 이어받아 새 시간만 다시 찾게 하고, **예약 취소**는 확인 다이얼로그를 한 번 거치는 되돌릴 수 없는 동작으로 취급합니다. 두 동작 모두 참석자 전원에게 알림이 전송됐다는 사실을 화면에서 바로 확인할 수 있습니다.

### 나의 회의 프로필
10. **기본 정보·업무 패턴 설정** — 이름·소속팀·프로필 사진, 업무시간·점심시간·집중 업무 시간, 선호 시간대, 피하고 싶은 시간, 회의 전후 이동시간 확보(버퍼)까지 한 화면에서 확인하고 수정합니다.
11. **캘린더 연동** — 구글 캘린더를 연동하면 외근·휴가·반복 회의가 자동 반영되고, **외근 전·후 각 최소 1시간은 회의 불가능한 시간**으로 자동 제외됩니다.

## 🧪 데모 시나리오 안내

이 프로토타입은 별도 백엔드 없이 더미 데이터로 동작합니다. 홈 화면에 떠 있는 알림이나 특정 입력값처럼 조건이 맞아야 나타나는 화면들을 아래에 정리했습니다 — 코드를 보지 않아도 이 표대로 조작하면 그대로 재현됩니다.

| 시나리오 | 재현 방법 | 확인 포인트 |
|---|---|---|
| 필참 초대 알림 | 홈 화면 "나에게 온 제안" 첫 번째 카드 | 초대받은 입장에서 후보 시간 중 가능한 시간을 골라 응답하는 흐름 |
| 선택 참석 초대 알림 | 홈 화면 "나에게 온 제안" 두 번째 카드 | 이미 확정된 시간이라 시간 결정에는 관여하지 않고 참석 여부만 응답 |
| 회의 취소 알림 | 홈 화면 "나에게 온 제안" 세 번째 카드 | 내가 참석자였던 확정 회의가 조직자에 의해 취소된 경우 — "확인"을 누르기 전까지 계속 목록에 남아 있는 응답형 알림 |
| 추천 결과 없음 (빈 상태) | 회의 만들기에서 참석자 6명을 모두 추가하고 전원을 필참으로 지정 후 "MeetFit 추천받기" | 필참자가 많을수록 모두가 가능한 시간을 찾기 어렵다는 전제를 빈 상태 화면으로 표현. 실제 추천 엔진은 고정된 3개 후보만 채점하는 프로토타입이라 진짜 일정 충돌로 0건이 나오지는 않으며, 필참 5명 이상을 데모용 트리거로 사용 |
| 예약 취소 | 확정된 회의 상세 화면 > 예약 취소 | 확인 다이얼로그 → 참석자 전원에게 취소 알림을 보냈다는 토스트까지 확인 |
| 일정 변경 | 확정된 회의 상세 화면 > 일정 변경하기 | 확인 다이얼로그 → 참석자 전원에게 변경 알림을 보냈다는 토스트, 제목·참석자·목적은 그대로 이어받아 새 시간만 다시 탐색 |

## 📐 적합도 점수는 이렇게 계산돼요

100점 만점을 6개 요소로 쪼개서 더합니다.

| 요소 | 기본 배점 | 의미 |
|---|---|---|
| 필참 참석 | 40 | 필참자 전원이 참석 가능한가 |
| 참석률 | 20 | 전체 인원 중 몇 명이 가능한가 |
| 개인 선호 | 15 | 각자의 선호 시간대와 얼마나 맞는가 |
| 업무 맥락 | 15 | 점심 직후·집중 업무 시간·외근 전후 버퍼와 겹치는가 |
| 여유 시간 | 5 | 회의 앞뒤로 얼마나 여유가 있는가 (피로도) |
| 이동 시간 | 5 | 자리 이동·온라인 전환에 필요한 버퍼가 있는가 |

* 이 배점은 고정이 아니라 **회의 목적에 따라 가중치가 달라집니다.** 예를 들어 '의사결정'은 필참 참석 배점을 1.8배로 키워 필참자 전원 참석을 최우선하고, '문제 해결'은 업무 맥락·여유 시간 배점을 키워 몰입 가능한 여건을 우선합니다. '긴급 대응'은 아예 점수보다 **가장 빨리 모일 수 있는 시간**을 우선합니다.
* 여러 후보의 점수가 동점이면, 굳이 미룰 이유가 없다는 원칙으로 **더 이른 시간을 자동으로 우선** 추천합니다.

## 💻 프로토타입 구현 및 기술 스택

본 과제는 UI 디자인에 그치지 않고, 사용자가 실제로 탭을 전환하며 조건별 점수의 변동과 추천 근거를 직관적으로 확인할 수 있도록 High-fidelity 웹 프로토타입으로 제작되었습니다.

* **Tech Stack:** Vanilla HTML5, CSS3 (CSS Variables 기반 Toss-style 디자인 시스템 구현), Vanilla JavaScript (ES6+)
* **UX Detail**
  * 마우스 호버(Hover) 시 부드러운 인터랙션 트랜지션 제공 및 은은한 카드 그림자 구현
  * 필수/선택 참석자 토글 상태 변경 시 추천 점수 카드가 동적으로 연동되는 마이크로 인터랙션 구현
  * 필참/선택 참석자 응답 진행률을 프로그레스 바로 실시간 반영, 미응답자 리마인드 발송 플로우 구현
  * 내가 주최자가 아닌 회의(초대받아 참석만 하는 경우)는 추천 근거·일정 변경 등 주최자 전용 UI를 자동으로 숨겨, 역할에 따라 다른 정보만 노출
  * 응답이 시작된 제안은 수정할 수 없도록 막고 취소만 허용하는 등, 실제 조율 과정에서 생기는 예외 상황까지 화면 상태로 설계
  * 프로필 사진은 파일 선택 즉시 미리보기로 반영되고, "저장하기"를 눌러야 사이드바 아바타까지 최종 반영되는 등 다른 프로필 필드와 저장 시점을 일치시킴
  * 일정 변경(재조율) 시 참석자·회의 목적뿐 아니라 제목·장소까지 그대로 이어받아, 처음부터 다시 입력하지 않도록 함
