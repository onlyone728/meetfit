# 🗓️ meetfit — 복잡한 맥락을 계산해 정답을 알려주는 회의 예약 솔루션

> **2026 Toss Product Design Challenge** > "6명의 동료가 1시간 회의 일정을 잡는 경험을 설계해주세요."

많은 사람이 일정을 잡을 때 기존 캘린더 앱은 단순히 '비어 있는 시간'만 보여줍니다. 하지만 실제 협업 환경에서는 필수/선택 참석자 구분, 점심 직후 피로도, 외근 이동 시간 등 **눈에 보이지 않는 개인의 맥락(Context)을 인간이 수동으로 계산**해야 합니다.

`meetfit`은 무겁고 번거로운 수동 스케줄러 페이지를 과감히 제거하고, 연동된 데이터와 주관적 성향 프리셋을 계산하여 **모두에게 가장 쾌적한 최적의 의사결정(Decision First) 시간대를 점수(Score)로 제안**하는 툴입니다.

---

## ✨ 핵심 기능 및 화면 설계 (Core UX)

### 01. 회의 프로필 프리셋 (Meet Profile)
* 사용자가 매번 번거롭게 캘린더에 반복 일정을 그릴 필요가 없습니다. 기본 사내 캘린더(Google, Outlook) 데이터가 실시간 연동됩니다.
* 사용자는 캘린더가 알 수 없는 **인간의 주관적 선호 맥락(집중 업무 시간, 기피 시간, 선호 시간)**만 탭 몇 번으로 가볍게 등록해 둡니다.

### 02. 적합도 스코어링 시스템 (Match Score)
* 기존 서비스의 흑백논리(가능/불가능)에서 벗어나, 현실의 '회색 지대(되긴 하는데 피곤한 시간 등)'를 **정량화된 점수(ex. 96점)**로 치환하여 인지 부하를 줄였습니다.
* **회의 전후 여유 시간 (피로도 제어):** 추상적인 피로도 개념 대신, '회의 앞뒤로 최소 1시간의 빈 블록이 확보되었는가'를 정량 규칙으로 삼아 점수에 반영하고, 직관적인 카피로 근거를 노출합니다.

### 03. 주간 기간(Range) 우선 추천 디폴트
* 실무에서 6명의 일정을 잡을 때 특정 일을 고집하기보다 "다음 주 중 편한 시간"으로 접근하는 맥락이 더 큽니다.
* 일자별 탐색 대신 **[기간으로 찾기]**를 디폴트로 하여, 일주일 치 데이터 중 모두가 가장 쾌적한 '최적의 하루와 1시간 슬롯 TOP 3'를 먼저 스포트라이트 형태로 보여줍니다.

## 📝 과제 질문 3가지 답변

### Q1. 사용자의 어떤 문제를 발견했나요? 그것이 왜 문제라고 생각했나요?
* **[문제 발견] 맥락적 피로감과 의사결정의 지연** 여러 명의 일정을 잡을 때 주최자는 단순히 '캘린더의 빈칸'만 찾는 것이 아니라, "누가 필수인지", "점심 직후라 피곤하진 않을지", "이동 시간이 부족하진 않은지" 등 보이지 않는 개인의 맥락을 머릿속으로 수동 계산해야 합니다.
* **[이유]** 인원이 6명만 되어도 경우의 수가 기하급수적으로 늘어나며, 빈 시간을 겨우 찾아 제안해도 누군가의 기피 시간과 겹쳐 일정 조율이 도돌이표처럼 반복됩니다. 이는 기존 캘린더 툴의 구조적 한계로 인해 발생하는 극심한 인지적 과부하와 커뮤니케이션 비용입니다.

### Q2. 그 문제를 어떻게 해결했나요?
* **[결과 중심의 의사결정 뷰 (Decision First)]** 사용자가 복잡한 6명의 캘린더를 겹쳐 보며 고민하지 않도록, 시스템이 동료들의 캘린더 데이터와 회의 프로필 프리셋을 기반으로 최적의 시간대를 점수(Score)화하여 Top 추천 슬롯으로 제안합니다.
* **[조건의 가중치 자동 계산]** 필수 참석자의 시간은 절대적인 기준으로 반영하되, 선택 참석자의 기피 시간이나 개인별 선호도(예: 점심 직후 피하기)는 점수 가중치에 유연하게 반영하여 '모두에게 가장 무리 없는 시간'을 1초 만에 찾아낼 수 있도록 인터랙션을 단축했습니다.

### Q3. 왜 이 방식으로 설계했나요?
* **[비어 있는 시간과 좋은 시간은 다르기에]** 기존 서비스처럼 '시간 기준'으로 빈칸을 보여주는 것이 아니라, '사람 중심'의 맥락을 점수라는 시각적 직관성으로 치환했습니다. 시스템이 제안한 추천 시간을 보는 순간 사용자는 복잡한 계산 없이 "이 시간으로 예약" 버튼 하나만 누르면 끝나는 Zero-cost 경험을 제공하고 싶었습니다.
* **[불필요한 수동 스케줄러 페이지의 제거]** 사내 연동된 데이터와 초기 프로셋 설정만으로 시스템이 보이지 않는 곳에서 계산을 끝내기 때문에, 사용자는 화면을 가득 채우는 무거운 달력 화면을 마주할 필요가 없습니다. 오직 **'동료 선택 ➡️ 최적 시간 확인 ➡️ 예약'**이라는 목적에만 집중된, 토스 고유의 단순함(Simplicity) 가치를 극대화하기 위해 이와 같이 설계했습니다.

## 💻 프로토타입 구현 및 기술 스택

본 과제는 UI 디자인에 그치지 않고, 사용자가 실제로 탭을 전환하며 조건별 점수의 변동과 추천 근거를 직관적으로 확인할 수 있도록 고충실도(High-fidelity) 웹 프로토타입으로 제작되었습니다.

* **Tech Stack:** Vanilla HTML5, CSS3 (CSS Variables 기반 Toss-style 디자인 시스템 구현), Vanilla JavaScript (ES6+)
* **UX Detail:** * 마우스 호버(Hover) 시 부드러운 인터랙션 트랜지션 제공 및 은은한 카드 그림자 구현
  * 필수/선택 참석자 토글 상태 변경 시 추천 점수 카드가 동적으로 연동되는 마이크로 인터랙션 구현
